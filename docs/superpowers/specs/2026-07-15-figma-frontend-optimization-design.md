# 知构软考首批前端优化设计

日期：2026-07-15

状态：第一版已实施；2026-07-15 修订方案已确认，待二次实施

范围：首页、全局导航、真题列表

## 1. 背景与目标

本轮优化参考 Figma Make 文件 `aisoft` 的首页和真题列表页，优先完成“刷真题”主流程。打基础与论文冲刺暂不开放；试卷目录允许游客浏览，只有开始或继续刷题及访问个人数据时要求登录。

固定设计基准：[Figma Make `aisoft`](https://www.figma.com/make/SWQfsGunfx5VC8yrxALnDr/aisoft?p=f&t=psDu1cudT29SZrXY-0)，版本 28，页面路径 `/home` 与 `/papers`。桌面视觉基准为 1366×768；响应式另外按第 8 节的四个宽度验收。Figma MCP 在当前环境未配置，因此实施以该版本的可见预览和本规格的唯一化决策为准。

目标：

- 首屏清楚表达“真题优先”的产品价值，并提供明确的试卷目录入口。
- 全局导航在桌面端与移动端均可用，登录态和现有菜单功能保持不变。
- 真题页按照“功能入口、筛选、试卷列表”的顺序组织信息，游客也能查看真实试卷排列。
- 真实 API 的加载、失败和空数据状态有完整反馈；开始会话仍复用现有错误处理。
- 后续页面可复用本轮形成的颜色、间距、圆角、阴影和组件模式。

## 2. 设计原则

采用“高保真参考 + 生产化修正”：

- 高保真参考 Figma 的浅蓝页面背景、白色卡片、蓝色主操作和宽松留白；刷真题是唯一可用的学习路线，其他路线明确表现为暂停开放。
- 不照搬 Figma Make 中的 mock 数据或重复页面；试卷、进度状态、用户统计等业务数据来自项目现有状态和 API。品牌说明、备考方法等运营文案允许作为明确的静态内容存在，但不得伪装成实时统计。
- Figma 中“刷题记录、错题分析”的入口复用现有路由 `/practice-history` 和 `/wrong-questions`，游客点击时先进入登录页。
- 不直接修改 `src/components/ui/` 下的 Shadcn 组件。
- 优先使用标准 Tailwind 类；确有设计色无法稳定表达时再使用 CSS 变量或行内样式。

## 3. 信息架构

### 3.1 全局导航

桌面端从左到右为：

1. 品牌标识“知构软考”，点击返回首页。
2. 主导航：打基础、刷真题、论文冲刺；刷真题是唯一链接，打基础与论文冲刺使用不可点击文本，降低对比度并显示“暂未开放”。
3. 考试倒计时。
4. GitHub Star 入口。
5. 登录/注册按钮，或已登录用户头像与菜单。

当前路由统一使用蓝色实底、白色文字高亮，避免仅依赖文字颜色。导航固定在页面顶部，滚动时保留轻微边框与背景模糊。

桌面端和移动端的停用入口均不得响应点击或键盘激活，使用 `aria-disabled="true"`，不使用 `Link`。现有 `/foundation`、`/essay-sprint` 及论文相关页面代码和路由保留供后续开发，但不作为正常用户入口。

小于 1024px 时保留品牌、倒计时简版和菜单按钮；主导航、GitHub、登录态操作收进基于现有 Shadcn `Dialog` 定位实现的右侧面板。打开后焦点进入面板，Tab 焦点不能逃逸，关闭按钮、点击导航项或 Escape 均可关闭，关闭后焦点回到菜单按钮。

### 3.2 首页

首页按以下顺序组织：

1. **首屏 Hero**：主标题和说明强调“真题优先”；主按钮“查看历年真题”进入 `/papers`，次按钮“了解刷题方式”滚动到本页 `#learning-paths`。
2. **三条学习路线**：继续展示打基础、刷真题、论文冲刺三张卡片以保留产品路线，但只有刷真题使用链接语义并进入 `/papers`；其他两张卡降低饱和度、去掉悬停位移，显示“暂未开放”且不可点击。
3. **作者经历**：保留真实备考路径与三科通过结果，压缩装饰与文案宽度，作为信任证明而非第二个 Hero。
4. **方法说明**：优先说明真题筛选、练习/考试双模式和错题复盘三个刷题能力。
5. **产品能力摘要**：使用“历年真题”“双模式刷题”“错题沉淀”，不宣传当前暂停开放的 AI 论文能力。

桌面首屏应在常见 1366×768 视口内同时露出 Hero 主操作和三条路线卡片上缘。移动端所有内容改为单列，按钮在窄屏下可纵向排列。

### 3.3 真题页

页面按以下顺序组织：

1. 页面标题“刷真题”。
2. 功能导航：试卷列表、刷题记录、错题分析。试卷列表对所有人开放；已登录时后两项是跨路由链接，并分别显示 `fetchPracticeHistory({ pageSize: 1 }).total` 的会话总数与 `fetchWrongQuestions().total` 的错题条目总数；游客看到锁定状态，点击调用 `onShowAuth`。
3. 白色筛选卡：科目筛选、题型分类两组胶囊按钮。
4. 当前筛选标题和试卷数量。
5. 响应式试卷卡片列表。

试卷卡内容包括：

- 年份与月份标题；
- 科目与题型标签；
- 题目数量；已登录时额外展示个人练习次数 `paper.practiceCount`，游客不展示该字段；
- 已登录时可展示进行中状态标签；游客不展示个人状态。本轮不显示数值进度，因为现有接口的 `completedCount/progress` 尚不是可靠的当前会话进度；
- 更新时间；
- 游客统一显示“开始刷题”，点击后进入登录页；已登录用户根据状态显示“开始刷题”“继续刷题”或“重新刷题”。

试卷卡在 768px 及以上使用两列，小于 768px 使用单列。筛选项允许横向换行，不出现横向页面滚动。

## 4. 视觉系统

### 4.1 色彩

- 页面背景：`slate-50` 到浅蓝色的轻渐变，真题内容区以 `blue-50/40` 为主。
- 主色：`blue-600`，悬停使用 `blue-700`。
- 主文字：`slate-900`；正文：`slate-600`；辅助文字：`slate-500`。
- 路线识别：打基础 `blue-*`、刷真题 `emerald-*`、论文冲刺 `violet-*`。
- 倒计时：`amber-50` 背景配 `amber-800` 文本。
- 错误、空状态、成功状态使用标准 `red-*`、`slate-*`、`emerald-*`。

### 4.2 版式与组件外观

- 内容最大宽度统一为 `max-w-7xl`，左右边距在移动端为 16px，桌面端为 24–32px。
- Hero 标题桌面端约 48–56px，移动端约 36–40px，控制在两行以内。
- 主要卡片圆角使用 16–20px，按钮使用 10–12px；避免每个区域使用不同圆角。
- 卡片默认使用细边框与轻阴影，悬停只做小幅上移、边框加深和阴影增强。
- 交互动画控制在 150–250ms，并遵守 `prefers-reduced-motion`。

## 5. 组件边界

### 5.1 `AppHeader`

职责：品牌、启用/停用主导航、倒计时、GitHub 入口、登录状态和移动端菜单。

依赖：React Router、`useAuth`、现有 Avatar/Dropdown/Button 组件。允许 `useAuth` 新增只表示首次认证检查完成的 `isAuthInitialized` 状态；该状态不改变 Token、用户数据或登录规则。

不负责：页面内容、后台权限判断或用户数据请求。

### 5.2 `LearningLanding`

职责：首页各内容区的结构、文案与导航动作。

建议在文件内部或独立轻量组件中拆分 `HeroSection`、`LearningPathCard`、`AuthorStory` 和 `CapabilityStrip`，避免主组件继续增长。

不新增服务端请求；点击动作继续通过 React Router 导航。

### 5.3 `PapersPage`

职责：为所有访客加载真题目录；仅为已登录用户加载个人统计；管理筛选状态、渲染功能导航和试卷卡，并在开始刷题前执行登录门控。

保持现有 `fetchPapers`、`fetchPracticeHistory`、`fetchWrongQuestions` 调用和 `onStartPaper` 回调。`fetchPapers` 对游客和用户都调用；后两项只在登录后调用。允许抽取纯展示的 `PaperCard`、`FilterGroup` 和 `PapersPageNav`。`ExamPaper` 类型位于 `src/types/exam.ts`，`PapersPage` 不从 `src/data/examPapers.ts` 导入筛选常量或业务类型。

### 5.4 后端试卷目录

`GET /paper/list` 改为可选认证：无 `Authorization` 返回纯目录；有合法 Token 返回目录和个人会话状态；有非法或过期 Token 仍返回真实 HTTP 401。这里的“合法 Token”同时要求 JWT 签名和有效期正确、用户存在、未删除且已启用。可选认证和强认证复用同一个活动用户校验。

目录无论匿名还是登录均只返回 `is_deleted = 0` 且 `publish_status = 1` 的试卷；未发布试卷只能通过管理端接口查看。`PaperServiceImpl` 在 `userId == null` 时不得查询个人会话表，直接使用空会话集合构造 DTO。

试卷目录使用专用 DTO。公共字段白名单为 `id`、`name`、`subjectName`、`paperCateId`、`paperYear`、`paperMonth`、`questionTotal`、`updateTime`；登录用户额外获得 `readCt`、`doingSessionId`、`paperStatus`。游客响应必须省略个人字段，而不是仅依赖前端隐藏；`frontMockId`、`isDeleted`、`createTime`、`completedCount`、`progress` 等内部或不可靠字段不进入目录 DTO。

`GET /paper/detail/{paperId}`、`GET /question/{questionId}` 和 `POST /session/start` 继续强制活动用户认证。`/paper/detail` 和 `/session/start` 还必须验证目标试卷未删除且 `publish_status = 1`；普通用户直接请求未发布试卷时返回 404。三个接口只返回或装载 `question.is_deleted = 0` 的题目；`/question/{questionId}` 必须同时满足题目未删除且至少关联一套未删除、已发布试卷，否则返回 404。管理端预览使用独立管理接口。`/question/{questionId}` 不再允许游客通过 `withAnswer=true` 获取答案；目录公开不等于题目和答案公开。

`/paper/list` 响应统一设置 `Cache-Control: private, no-store`，禁止代理、CDN 或浏览器共享包含个人状态的响应。

## 6. 数据流与状态

### 6.1 首页与导航

- 路由状态决定当前主导航高亮。
- `useAuth` 决定显示登录按钮或用户头像菜单。
- 倒计时由 `AppHeader.tsx` 中唯一常量 `2026-05-23T00:00:00+08:00` 计算；按该固定时区比较，当前时间达到或超过考试日后统一显示“本期考试已结束”，不显示 `0 天`，也不推测下一考试日期。

### 6.2 真题页

1. `useAuth` 在首次 `checkAuthStatus()` 的同步/异步分支全部结束后将 `isAuthInitialized` 设为 `true`。`PapersPage` 等待该值后始终请求 `fetchPapers()`；这样携带的 Token 要么已确认有效，要么请求按游客身份发送。
2. 已登录时并行请求 `fetchPracticeHistory({ pageSize: 1 })` 和错题统计；游客不发送这两项请求。试卷请求控制主要加载与错误状态，个人统计失败不阻断试卷列表，对应计数固定显示 `--`。
3. 科目选项由 `fetchPapers()` 返回值中的非空 `subject` 去重生成；已知科目按“系统架构设计师、系统分析师、软件设计师、网络工程师、数据库系统工程师、信息系统项目管理师”排列，未知科目使用 `localeCompare('zh-CN')` 后追加。题型选项由当前科目下的非空 `category` 去重生成，按“综合知识、案例分析、论文”排列，未知题型同样按中文名称追加。
4. 初次加载选择第一个真实科目及该科目的第一个真实题型，不提供“全部”项，以匹配 Figma。科目切换后，如果当前题型在新科目下不存在，立即重置为该科目的第一个真实题型。没有任何试卷时不设置选择值，进入全局空状态。
5. 游客点击任意试卷“开始刷题”时只调用 `onShowAuth`，不打开模式弹窗、不调用 `onStartPaper`。已登录时复用现有模式选择和开始/继续会话逻辑。现有 `completedCount/progress` 不参与展示；`status === 'in_progress'` 只显示“进行中”和“继续刷题”。
6. 请求层抛出包含 HTTP `status` 和响应体业务 `code` 的结构化错误。只有真实 HTTP 401 或业务码 401 才清除本地 Token 与用户；网络错误和 5xx 保留认证信息并进入可重试的认证检查失败态。`/paper/list` 在请求期间收到 401 时，前端先清除本地认证，再自动按游客身份重新请求目录；可显示“登录已过期，已切换为游客浏览”的非阻塞提示，但不跳转登录。服务端不把无效 Token 静默当成游客。
7. 认证初始化使用唯一状态转换：首次检查和每次重试开始时设置 `isAuthInitialized = false`、清空旧的 `authInitializationError`；成功时更新用户并设置 `isAuthInitialized = true`；401 时清除 Token/用户并设置 `isAuthInitialized = true`，随后 `/papers` 按游客加载目录；网络错误或 5xx 保留 Token/用户，保持 `isAuthInitialized = false` 并设置 `authInitializationError`。`PapersPage` 在该错误存在时不发送目录或个人请求，显示“认证状态检查失败”和重试按钮。重试必须先回到检查中状态，检查完成前不能发业务请求。

## 7. 异常与边界状态

- **加载中**：统一使用与最终布局一致的筛选卡骨架和四张试卷卡骨架，避免页面跳动。
- **试卷请求失败**：显示错误原因和“重新加载”按钮；点击后重新执行试卷请求，登录用户同时重新请求历史总数和错题总数。
- **无试卷**：接口整体为空时显示“暂时还没有可用试卷”。由于题型选项只从当前科目的真实试卷生成，不设计不可到达的“筛选组合为空”状态。
- **统计请求失败**：不影响试卷主流程，不弹阻塞式提示；登录用户的对应导航计数显示 `--`。游客固定显示锁定图标而非数字。
- **开始会话失败**：保留 `App.tsx` 现有 `alert` 行为和错误内容；Toast 改造、请求中禁用和失败后恢复模式弹窗明确不属于本轮完成标准。
- **长科目名**：筛选按钮允许自适应宽度与换行，卡片标题不得溢出。
- **考试日期过期**：固定显示“本期考试已结束”。
- **认证异常**：缺失签名、错误签名、过期、格式错误、缺少必要载荷、用户不存在、用户已删除或禁用均归一为 `UnauthorizedException`，由全局异常处理返回真实 HTTP 401 和响应体 `code: 401`；不能落入 HTTP 200 或通用 500。

## 8. 可访问性与响应式

- 所有可点击卡片使用真实按钮或链接语义，不能只依赖 `div onClick`；停用路线卡使用非交互容器和 `aria-disabled="true"`。
- 图标按钮提供中文 `aria-label`；装饰图标对读屏隐藏。
- 焦点样式清晰，颜色对比达到 WCAG AA 的常规文本要求。
- 当前主导航和功能导航中真实路由使用链接及 `aria-current="page"`，不使用 `role="tab"`。游客的“刷题记录”“错题分析”锁定入口使用真实按钮调用 `onShowAuth`，不伪装成链接。
- 移动菜单和模式选择均复用 Shadcn `Dialog`；移动菜单将内容定位为右侧面板，不修改其 `ui/` 实现。两者打开后焦点进入面板、Tab 不逃逸、Escape 关闭、关闭后焦点返回触发按钮。
- 360px：移动导航、单列 Hero 按钮、单列路线卡和试卷卡；768px：移动导航、三列路线卡、两列试卷卡；1024px 与 1366px：桌面导航、三列路线卡、两列试卷卡。
- 四个验收宽度均需检查可见焦点、WCAG AA 常规文字对比度、`prefers-reduced-motion` 下无非必要位移动画，以及页面无横向滚动。

## 9. 测试与验收

### 9.1 自动检查

- `npm run build` 必须通过。
- 若项目补充类型检查或 lint 脚本，必须无新增错误。
- 浏览器控制台不得出现新增运行时错误或资源 404。

### 9.2 手工流程

1. 未登录访问首页，验证“查看历年真题”进入 `/papers`、“了解刷题方式”滚动到 `#learning-paths`；打基础和论文冲刺显示“暂未开放”且鼠标、键盘均不可激活，刷真题进入 `/papers`。
2. 已登录时头像菜单、个人中心、后台入口和退出行为不回归。
3. 在四个验收宽度验证导航布局；移动菜单和模式选择 `Dialog` 均完成打开后聚焦、Tab 圈定、Escape 关闭、关闭后焦点恢复。
4. 未登录直接访问 `/papers` 时只请求 `/paper/list`，能完成动态科目和题型筛选；不得请求历史与错题接口。
5. 游客点击试卷“开始刷题”只进入登录页，网络中不得出现 `/session/start`；已登录用户的开始、继续、重新刷题仍调用原有流程。
6. 游客点击刷题记录与错题分析进入登录页；已登录用户进入对应现有路由并显示真实计数。
7. 完成一条答题回归链：登录 → 打开试卷 → 分别验证练习/考试模式入口 → 作答并保存 → 刷新后继续 → 交卷 → 查看结果 → 重新刷题；开始会话请求失败时现有错误提示仍出现，且已有的当前会话状态不被前端清空。服务端幂等与网络不确定状态下的重复会话不属于本轮保证。
8. 分别复现主请求失败、登录用户统计失败、游客空目录、超长科目名和过期倒计时，确认唯一文案与布局；登录成功场景中的练习次数必须与 `paper.practiceCount` 一致，游客不显示练习次数和个人状态。
9. 后端验证 `/paper/list`：无 Header 返回 HTTP 200 且只有公共字段；两个具有不同会话数据的合法用户分别返回 HTTP 200 和各自个人字段，不能串号；非法、过期、已删除、已禁用用户 Token 返回真实 HTTP 401 和响应体 `code: 401`。所有响应包含 `Cache-Control: private, no-store`，未发布试卷对游客和普通用户均不可见。
10. 验证 `/paper/detail/{paperId}`、`/question/{questionId}?withAnswer=true` 和 `/session/start` 在无 Token、非法 Token 时均返回真实 HTTP 401；普通活动用户直接请求未发布试卷详情、开始未发布试卷或读取只属于未发布试卷的题目时返回 404；直接读取已软删除题目返回 404，试卷详情和新会话不包含已软删除题目；合法活动用户对已发布内容的现有调用不回归。
11. 前端分别验证初始非法 Token、使用期间 Token 到期、`/auth/me` 网络失败和 `/auth/me` 5xx：前两项清除认证并按游客留在真题目录，点击刷题后才进入登录；后两项保留认证信息、显示重试态且不发送目录请求。重试期间检查 `isAuthInitialized = false`，不得提前发送业务请求。
12. 对照固定的 Figma 版本 28 `/home` 与 `/papers` 预览，检查首页和真题页的主色、间距、卡片层级和主操作位置。

## 10. 非目标

本轮不包含：

- 登录、个人中心、答题页、结果页、错题页和论文页的全面重设计；
- 数据库结构调整；除修复 `/paper/list`、`/paper/detail`、`/question`、`/session/start` 和认证 HTTP 状态所必需的安全边界外，其他业务鉴权规则调整；允许 `/session/start` 增加“活动用户且目标试卷未删除、已发布”的校验。
- 新增统计 API；
- 修复后端试卷进度口径或在本轮展示数值进度；
- 重构开始会话的 `alert`、按钮禁用和弹窗错误恢复；
- 新增图标库或替换 Shadcn 基础组件；
- 将 Figma Make 的 mock 数据复制进生产代码。

## 11. 完成标准

- 首页、导航和真题列表形成一致的视觉系统，整体观感与 Figma 参考稿匹配。
- 打基础与论文冲刺在桌面端和移动端均明确不可选；刷真题是唯一主路线。
- 游客可浏览真实试卷目录和筛选结果，但不能创建会话或访问个人记录。
- 现有真实数据、登录态、动态筛选和完整答题流程无回归；页面不使用 mock 筛选常量，也不展示不可靠的数值进度。
- 360px 到 1366px 的关键宽度无横向滚动、内容遮挡或无法操作的控件。
- 加载、失败、空数据和已过期倒计时都有可理解的展示。
- 生产构建成功，并完成浏览器截图对照验证。
